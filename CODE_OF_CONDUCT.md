# Code de conduite

Serein s’engage à construire une communauté calme, respectueuse et inclusive.

Ce code de conduite s’applique aux discussions du dépôt, issues, pull requests, documentation, espaces communautaires liés au projet et toute interaction où quelqu’un représente Serein.

Ce document est adapté du Contributor Covenant et ajusté pour ce projet.

## Notre engagement

Nous voulons que la participation à Serein soit une expérience sans harcèlement pour tous, indépendamment de l’âge, du handicap, de l’ethnicité, des caractéristiques sexuelles, de l’identité et expression de genre, du niveau d’expérience, de l’éducation, du statut socio-économique, de la nationalité, de l’apparence personnelle, de la race, de la religion ou de l’identité et orientation sexuelle.

Nous nous engageons à agir de manière à soutenir une communauté ouverte, accueillante et saine.

## Comportements attendus

Exemples de comportements positifs :
- utiliser un langage respectueux et inclusif ;
- être constructif dans les retours et désaccords ;
- présumer la bonne foi tout en étant clair sur les préoccupations ;
- se concentrer sur ce qui est meilleur pour les utilisateurs et la communauté ;
- respecter les limites, la vie privée et le consentement ;
- accepter les retours avec grâce.

## Comportements inacceptables

Exemples de comportements inacceptables :
- harcèlement, intimidation ou discrimination ;
- attaques personnelles, insultes ou commentaires dégradants ;
- trolling ou comportement délibérément inflammatoire ;
- publication d’informations privées d’autrui sans permission explicite ;
- langage sexualisé ou attention sexuelle non sollicitée ;
- communication hostile, répétitive ou manipulatrice ;
- doxxing, menaces ou comportement abusif en public ou privé.

## Attentes spécifiques pour ce projet

Parce que Serein concerne la méditation et le bien-être émotionnel, nous demandons aux contributeurs d’être particulièrement attentifs au ton.

Veuillez éviter :
- les allégations médicales ou thérapeutiques sans base claire ;
- la culpabilisation des utilisateurs pour inconstance, stress ou anxiété ;
- l’introduction de tactiques de croissance coercitives dans les discussions ;
- le rejet des préoccupations de confidentialité comme secondaires.

## Responsabilités d’application

Les mainteneurs du projet sont responsables de clarifier et d’appliquer ce code de conduite.

Ils peuvent supprimer, modifier ou rejeter des commentaires, commits, issues, pull requests ou autres contributions non alignées.

## Périmètre

Ce code de conduite s’applique :
- aux issues, pull requests et discussions GitHub ;
- à la documentation et revues du projet ;
- aux chats ou appels liés au projet si créés ultérieurement ;
- quand quelqu’un représente publiquement le projet.

## Signalement

En cas de comportement inacceptable, signalez-le **privément** au mainteneur.

Canaux recommandés :
- utilisez le signalement de vulnérabilités privées GitHub si activé ;
- sinon, contactez **serein@cesarbroche.fr**.

À inclure dans le signalement :
- ce qui s’est passé ;
- où ça s’est passé ;
- quand ça s’est passé ;
- liens ou captures si pertinents ;
- si urgence de sécurité.

Si le signalement concerne le mainteneur, utilisez un contact alternatif si publié.

## Processus d’application

Les mainteneurs viseront à :
- accuser réception ;
- examiner de manière équitable et discrète ;
- protéger au maximum la vie privée du signalant ;
- prendre une action proportionnée.

Réponses possibles :
- avertissement privé ;
- demande d’arrêter un comportement ;
- restrictions temporaires ;
- suppression de contenu ;
- exclusion temporaire ou permanente.

## Bonne foi

Ce code de conduite protège la communauté, il ne cherche pas à étouffer les désaccords respectueux. La critique de bonne foi, le débat raisonné et l’opposition argumentée sont bienvenus.
